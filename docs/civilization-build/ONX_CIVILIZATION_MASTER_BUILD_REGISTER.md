## 1. Civilization Definition

**Civilization Name:** ONX Civilization
**Nature:** Complete institutional operating system for veterinary medicine
**Foundation Runtime:** onxos/onx-deploy
**Constitutional Basis:** ONX Constitutional Assets (onxos/onx-constitutional-assets)
**Forbidden Reference:** onxos/onx-intelligence-clean (out of scope until Atlas V6)

### Civilization Scope Statement

ONX Civilization is NOT merely the `onx-deploy` repository.

`onx-deploy` is the Platform Foundation Runtime. ONX Civilization is the complete build across 15 operational domains, 145 systems, and all implementation units required to deliver a constitutionally verified, production-ready, Founder-approved veterinary institutional platform.

### Civilization Completion Condition

ONX Civilization is COMPLETE when and only when:
- All 145 registered systems have status `COMPLETE`
- All 500 Implementation Units (IU) have status `COMPLETE`
- All 9 Certification Gates have been passed
- Founder Approval has been issued for each domain
- The Final Completion Certificate has been signed

## 5. Implementation Units (IU) Registry

Every system is broken into exactly 4 standard Implementation Units. Each IU must be independently completed and evidenced.

### Standard IU Types

| IU Type | Code | Description |
|---|---|---|
| Schema | IU-SCH | Database schema file + tables + indexes + migrations |
| API | IU-API | tRPC router + all required procedures + input validation + RBAC |
| UI | IU-UI | Next.js App Router pages + components + forms + list/detail views |
| Test | IU-TST | Scripts in `scripts/` directory + passing evidence |

### IU Naming Format

`{SYS-ID}-{IU-TYPE}` e.g., `D02-S01-IU-SCH`, `D02-S01-IU-API`, `D02-S01-IU-UI`, `D02-S01-IU-TST`

### Foundation IUs (Priority 0 — Must Complete Before All Others)

These IUs are blocking all domain builds. They belong to D15 and D13 infrastructure.

| IU-ID | Description | Blocks | Status |
|---|---|---|---|

| FOUND-IU-03 | Branch-level RBAC extension (D15) | HR, Clinical, Finance staff scoping | NOT_STARTED |

## 12. Immutable Execution Rules

The following rules are IMMUTABLE and cannot be overridden by any executor or agent:

### Rule 1 — OCMBR is the Single Source of Truth
No execution item may exist outside OCMBR. If an item is not in this register, it does not exist and cannot be marked complete.

### Rule 2 — PR Reference Mandatory
Every Pull Request must reference at least one OCMBR IU-ID in its title and description. Format: `[{IU-ID}] {description}`. PRs without valid OCMBR references must not be merged.

### Rule 3 — Status from Evidence Only
No item may be marked COMPLETE without concrete, independently verifiable evidence. If evidence is missing, status must remain NOT_STARTED or PARTIAL.

### Rule 4 — No Forward Completion
Items in later waves cannot be marked COMPLETE before all Foundation IUs and all Wave N-1 items are COMPLETE. Wave order is enforced.

### Rule 5 — Constitutional Repository Boundary
- onxos/onx-deploy: All build work. Only repository where code is committed.
- onxos/onx-constitutional-assets: READ ONLY reference.
- onxos/onx-intelligence-clean: FORBIDDEN until Atlas V6 is formally authorised.

### Rule 6 — No Silent Assumptions
If scope, design, or dependency is unclear, halt and request Founder clarification. Do not assume and proceed.

### Rule 7 — Domain Isolation
Each domain's IUs must be built in isolation first (schema → API → UI → test) before cross-domain integrations are activated.

### Rule 8 — Quality Gate Non-Negotiable
`bun run lint`, `bun run build`, `bun run test` must pass before any PR is opened. These gates cannot be bypassed.

### Rule 9 — No Completion Calculation from Repository Files
Completion percentage is calculated ONLY from OCMBR IU statuses. Files existing in the repository are not evidence of completion unless the corresponding IU-TST evidence is present.

### Rule 10 — Civilisation vs Foundation Distinction
The Platform Foundation Runtime (authentication, SECH, Titan, etc.) is COMPLETE. It is NOT counted toward Civilisation completion percentage. Civilisation completion starts from 0% under OCMBR.
